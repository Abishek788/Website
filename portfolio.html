Portfolio Website on AWS with CI/CD, Custom Domain, SSL, and CloudFront Distribution

Project Overview

This project focuses on deploying a professional, secure, and highly available portfolio website on AWS to highlight cloud engineering expertise. The website utilizes key AWS services to achieve Continuous Integration and Continuous Deployment (CI/CD), custom domain integration, SSL/HTTPS, and global content delivery. This architecture exemplifies best practices in cloud deployment, emphasizing automation, security, scalability, and performance.

This document outlines the architecture of our scalable, serverless video transcoding platform built on AWS. The system efficiently handles video uploads, transcoding, and delivery while maintaining high availability and security.

Project Goals

  • Automate CI/CD Pipeline: Streamline code deployments through GitHub and AWS CodeBuild.
  • Interactive Contact Page: API-based serverless contact form integration using Lambda, API Gateway, and DynamoDB.
  • Custom Domain and DNS Management: Domain hosted and managed through AWS Route 53.
  • Enhanced Security with HTTPS: SSL encryption using AWS Certificate Manager.
  • Global Distribution and Caching: Content delivery optimized via AWS CloudFront.

Key Components and Implementation

1. Code Repository and CI/CD Pipeline

GitHub serves as the code repository, with AWS CodeBuild automating deployment by building and deploying changes on each commit to the main branch. The `buildspec.yml` file defines build commands, ensuring a seamless deployment process.

2. Contact Page with API Integration

The interactive contact form enables users to submit their details (name, email, message) securely. API Gateway serves as the RESTful API endpoint, triggering a Lambda function that processes and stores the data in DynamoDB for persistence and retrieval.

3. S3 as Static Website Hosting

Configured for static hosting with public access enabled to serve HTML, CSS, and JavaScript files. Versioning is enabled for data backup and recoverability.

4. Custom Domain Setup with Route 53

The custom domain is managed with Route 53, with an alias record pointing to CloudFront for efficient routing and DNS management.

5. SSL/HTTPS with AWS Certificate Manager (ACM)

SSL certificates issued by ACM, validated through DNS, and applied on CloudFront, ensure secure HTTPS traffic for enhanced data protection.

6. Global Content Delivery with CloudFront

CloudFront distributes content worldwide, using caching strategies for faster load times and Origin Access Identity (OAI) to secure S3 access.

Challenges and Solutions

  • Automating deployment from GitHub to S3: Solved with AWS CodeBuild.
  • Securing S3 content while using CloudFront: Implemented OAI for secure access.
  • Managing SSL certificate renewal: Automated with ACM for uninterrupted HTTPS.

Conclusion

This portfolio project demonstrates expertise in deploying secure, scalable, and automated infrastructure on AWS, showcasing proficiency in S3, CloudFront, Route 53, Lambda, DynamoDB, and CodeBuild for cloud-native deployments. This site serves as a testament to foundational cloud engineering skills.
